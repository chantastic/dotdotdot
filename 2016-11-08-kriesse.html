It sounds like we both want to help empower newcomers. I've only instructed React newcomers. So, my viewpoint is informed by those starting their career in React.

We have new JavaScript developers learning programming through the lens of components. Be it React, Angular, Ember, or Web Components, some new devs are learing UI development component-first. This is interesting. For as long as I've worked on the web we've attempted to emulate components by strategically stringing HTML, JS, and CSS together.

React developers don't touch HTML.
React Native developers don't touch HTML or CSS.
They only touch components.

For these new React Native developers, there is no cascade, no global styling scope, no specificity. It's just CSS declarations, right on a component.

Wasn't this kind what we were after all along?

I'm not a "CSS is dead" crier. I write A LOT of CSS. It doesn't have to "die" for CSS-in-JS to be valueable. Neither does it have to be universally adopted to be helpful.

There's a new class of JavaScript devs with no need for traditional CSS.

I stand by that.

I also stand by traditional developers. I don't think they need to be burdoned by exploratory JS toolsets.

The concept of "beginner" is becoming relative. There isn't just one entry point to frontend development. My concern is that we look aronud at what's happening and say "Hey you, you have to jump through the hoops as I did to be a 'real' developer." I don't agree with that.

If you can ship, you're a "real" dev.

That's my part. I'd love to hear what your position is on us crazy JAVASCRIPT-ALL-THE-THINGS folks :)
